DESCRIPTION: A distributed mnemonic synchronization game (Simon Says) designed for mobile-first rhizomatic networks.

01. CORE_MECHANICS

  • Mnemonic Sequence: The system generates a random index ($0-3$) and appends it to a global array.
  • Spatial Entropy: After stability reaches LEVEL_04, each correct input triggers a random 2D displacement ($X, Y$) to the interface nodes.
  • Signal Integrity: Uses Square-wave oscillators ($174\text{Hz}$ - $207\text{Hz}$) to ensure audio reproduction on restricted mobile hardware.

03. COLONY_PROTOCOL (MULTIPLAYER)

Handshake: Users must provide an identity starting with A or L followed by 8 digits.

The Capsule: When 4 players are registered, Slot 0 (the leader) triggers a 10-second sync timer. Once active, each player can only interact with their assigned node.

Cannibalization: If a player fails or disconnects, their ID is purged. The Slot 0 controller (CPU-mode) assumes authority over the vacant slot to maintain colony stability until one Supervisor remains.

04. OVERSEER_CONSOLE

The console.html serves as the master authority, allowing for:

  • Gate Locking: Prevents new logins when the capsule is at capacity or undergoing maintenance.
  • System Purge: Hard-deletes the `colony_session` node, terminating all active links.
  • Direct Broadcast: High-priority pink-text overrides injected directly into user terminals via the broadcast node listener.
