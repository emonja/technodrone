LOG DE CONSTRUCCIÓN: SEHNSUCHT

PROYECTO: Generador de Pulso Psicométrico Colectivo
ESTADO: Bloqueado / Operativo
VERSIÓN: 1.2.0 (Red Distribuida)

01. VISIÓN GENERAL

Desarrollo de una arquitectura de red para la generación de pulsos sonoros colectivos. El sistema ha evolucionado de un oscilador individual a un enjambre sincronizado donde el control se alterna entre la subjetividad del usuario y la autoridad de un mando maestro.

02. ESPECIFICACIONES TÉCNICAS

- Motor de Audio: Web Audio API (Sintetizador Sustractivo Monofónico)
- Backend: Firebase Realtime Database (Sincronía en < 100ms)
- Visualización: Canvas HTML5 (Persistent Ripple Ghosting)
- Interfaz: Estética Ámbar Terminal 07 (Baja contaminación cultural)

03. ARQUITECTURA DE DATOS (PSICO-MAPEO)

Pregunta de Usuario Parámetro Audio Configuración Técnica
P1: Tiempo Interno BPM (Tempo) 45bpm (Lento) | 80bpm (Normal) | 140bpm (Rápido)
P2: Textura Waveform / Freq Sine (440Hz) | Saw (110Hz) | Square (110Hz)
P3: Persistencia ADSR (Release) 0.2s (Efímero) | 1.5s (Persistente)

04. PROTOCOLO DE RED DISTRIBUIDA

Nodo Sujeto (index.html): Actúa como cliente receptor. Registra la matrícula del alumno, el estado de presencia ("online") y permite la interacción individual hasta recibir una señal de sobreescritura global.
Monitor de Colectivo (monitor.html): Visualización en tiempo real de todos los nodos activos. Clasifica usuarios por rol: Estudiante, Catedrático (IDs con 'L'), Sujeto L666 e Invitados.
Control Maestro (control.html): Mando de autoridad. Capaz de activar la sincronía global de frecuencia y tempo, silenciar el colectivo (Mute) o resetear el registro de asistencia de la base de datos.

05. NOTAS DE SEGURIDAD Y AUDIO

Se implementó el Protocolo de Invitados, permitiendo que cualquier ID con formato canónico acceda al sistema y sea monitoreado, incluso si no figura en la matrícula original. El ajuste de frecuencia para ondas senoidales (440Hz) garantiza que la presencia acústica sea perceptible en altavoces móviles de baja fidelidad.

11. INTEGRACIÓN DE ARCHIVO Y AMBIENTE SONORO

Interfaz de Acceso (structure.html): Se implementó un portal de entrada basado en la estética Technodrone, vinculando los cuatro nodos críticos: Nodo, Monitor, Maestro y Bitácora.
Handshake Audio: Se integró el soporte para sounds/hadshake.mp3, manteniendo la ambiencia original del repositorio Technodrone para la inicialización del sistema.
Persistencia de Datos: El sistema ahora gestiona roles diferenciados (Estudiante, Catedrático, Sujeto L666 e Invitados) con actualización de estado en tiempo real vía Firebase.
